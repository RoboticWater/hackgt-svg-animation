As you can see, we can now individually address elements of the SVG!

You can animate any element of this SVG just like it was an HTML element, though instead of <div/>'s, you're working with <path/>'s and <g/>'s.

<path/>'s define the shapes and lines which compose your SVG. If you look at the SVG code, you'll notice that we have two up top for the chains and 9 on the bottom for the letters (side note: if you're making an SVG with text, it's usually better to convert your text to paths than it is to leave it as text, because certain exporters will export the entire font with the logo, which has hella bytes).

<g/>'s are just groups, they're basically <div/>'s for SVGs.

There are other elements, but we won't get into them, just understand that SVGs can be addressed like any other HTML element, so whether it's <path/>'s or something else, it's all the same.

With this knowledge, you're probably pretty dangerous with SVGs right now. Most animations are just a bunch of simple transforms combined together to look good. I should state it here: good technical art is mainly just dedication. The skills aren't too difficult to grasp, it just takes practice to make better-looking stuff.

But the tutorial is not over; we still have stuff to learn!

Or stroke length.

I should pause on this one because we're not technically changing the length, it's actually a neat trick using the dashed tag.

Basically, imagine making a dash as long as the path itself, and then offseting that dash position by the same length. Because the gap between dashes is the length of the dash itself, it has the appearance of the path's length getting shorter. This article shows the process in-depth.

Now, thus far, we've been doing hover transitions, which are nice, but not exactly animations, so let's make an animation.

Animations are basically like transitions, but you have more fine grained control over how the transition progresses, and you can trigger it without hovering. They use @keyframes to define different movements in the animation.

I could've made it not repeat, but then you might have missed it.

Here, you can see how @keyframes work. You define the style of a particular element at a given % of completion, and then apply the keyframes as an animation. This is important: you don't define a keyframe at a time, but a percentage, so if you want to add more movements to the animation, note that you'll have to extend the time yourself to compensate.

Also note the transform-origin: Xpx Xpx; I put in the chain link CSS. This is because, by default the chain links will rotate around the entire shape's origin, which is the top left point.
